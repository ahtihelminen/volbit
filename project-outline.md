# Project Proposal: Stochastic Volatility Dynamics in Bitcoin via Coupled Langevin Equations

## 1. Introduction & Research Question
Standard financial models, such as Black-Scholes, assume that market volatility is a constant. This fails to account for "fat tails" (extreme events) and "volatility clustering" observed in highly speculative assets like Bitcoin. This project maps the physics of **Brownian motion in a potential** to the movement of financial markets.

**Scientific Question:** To what extent can a coupled system of Langevin equations (the Heston Model) replicate the non-Gaussian "stylized facts" of Bitcoin, and at what physical threshold (the Feller Condition) does the continuous-time stochastic approximation break down?

## 2. Theoretical Framework
We model the system as a particle (price) moving in a fluctuating environment (volatility) using two coupled Stochastic Differential Equations (SDEs):

1.  **Price Dynamics:** $$dS_t = \mu S_t dt + \sqrt{v_t} S_t dW_t^S$$
2.  **Variance Dynamics (Langevin):** $$dv_t = \kappa(\theta - v_t)dt + \xi \sqrt{v_t} dW_t^v$$

**Physics Mapping:**
* $\kappa(\theta - v_t)$: A **restoring force** (drift) pulling variance back to a long-term equilibrium $\theta$.
* $\xi \sqrt{v_t}$: The **volatility of volatility** (noise), representing the intensity of fluctuations.
* $\rho$: The **coupling constant** between price and variance Wiener processes.



## 3. Methodology & Technical Implementation
To satisfy the **Technical (30)** and **Research (30)** grading criteria, the project will focus on:

* **Numerical Integration:** Use the **Euler-Maruyama scheme** for the coupled SDEs.
* **Computational Efficiency:** Implementation using **NumPy vectorization** to simulate $10^5$ paths simultaneously, avoiding Python loops.
* **Data Calibration:** Parameters ($\kappa, \theta, \xi, \rho$) will be estimated using historical BTC/USDT hourly data to ensure the simulation grounded in reality.
* **Error Analysis:** Evaluation of the "Feller Condition" ($2\kappa\theta > \xi^2$). We will test if Bitcoin's extreme noise ($\xi$) causes the variance to hit zero, creating unphysical numerical instabilities.



## 4. Research Analysis & Expected Results
The project will analyze the following "stylized facts" generated by the Langevin dynamics:

1.  **Leptokurtosis (Fat Tails):** Comparing the simulated price return distribution against a Gaussian benchmark to show why Bitcoin is more "dangerous" than standard random walks.
    
2.  **The Leverage Effect Paradox:** Investigating the correlation $\rho$. While indices show $\rho < 0$, we will test the hypothesis that Bitcoin exhibits "speculative coupling" where $\rho$ can shift during bull runs.
3.  **The Volatility Smile:** Deriving the implied volatility surface from simulated option prices to show the model's departure from Black-Scholes.
    

## 5. Critical Discussion (Foundational Weaknesses)
* **The Continuum Fallacy:** Critique the use of smooth Langevin dynamics for an asset prone to discrete "jumps" (news-driven shocks).
* **Parameter Instability:** Discuss how the "equilibrium" $\theta$ is not a physical constant but is regime-dependent, limiting the model's predictive power.
* **The Feller Violation:** Analyze historical "flash crashes" where the model's mathematical foundations collapse due to extreme volatility.
